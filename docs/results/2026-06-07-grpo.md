# 2026-06-07 — New task: diameter (horizon-dip), first training run

The first NEW inference target beyond gravity. Breaks the spin/diameter-vs-g measurement
degeneracy (diameter was invisible to drop/pendulum since they only read effective g).

## The task
Infer the planet's **diameter (m)** via a new tool `measure_horizon_dip(height_m)`: from height
h the horizon dips by `alpha` where `cos(alpha)=R/(R+h)`, so `R=2h/alpha^2` (alpha in RADIANS) and
diameter `=2R`. Pure geometry — independent of g/spin/mass (verified). Spin deferred (the coupled
Foucault+latitude task is a separate, harder greenlight).

## How we got here (de-risked before training)
1. **Feasibility workflow**: physics verified simulatable from existing `WorldParams`; the deg→rad
   trap on `R=2h/alpha^2` is ~3283× (alpha squared), so the calculator gained sin/cos/radians and
   the briefing emphasizes "alpha in RADIANS".
2. **gpt-4o-mini benchmark**: 60% solve / 1.0% median rel-err — well-posed, reasoning tractable;
   the limiter was under-averaging (the skill GRPO teaches).
3. **Integration** (rotating `target`, `score_value`, `--target diameter`) + 25/25 unit tests +
   a clean end-to-end smoke.
4. Trained on the corrected **multi-world batching** (4 worlds/step), beta 0.005, lr 2e-6.

## Result — `rl-diameter-mw4` (250 steps, exit 0): SUCCESS
| | early | mid | late |
|---|---|---|---|
| physics | 0.21 | 0.32 | **0.46** |
| n_experiments | 1.66 | 3.09 | **4.98** (median 6) |
| %≥3 experiments | 18% | 83% | **98%** |

Final (last 20): physics **0.476**, reward **0.869**, tools 5.43, entropy 0.40, **max entropy 0.45
(no divergence)**. Late n_exp distribution peaks at **n=6 (53%)** + n=3 (27%) + n=5 (16%).

**Read:** the 1.7B learned the diameter task to essentially the **gravity task's level** (gravity
final ~0.48 / ~0.88), and — the key skill — **learned to aggregate** (~6 repeated dip measurements
averaged down) rather than answering off one noisy reading. The deg→rad/`alpha^2` hurdle did not
block it (calculator `radians()` + the briefing worked). Aggregation here (n≈6) is even higher than
the gravity multi-world run (n≈4.7), plausibly because diameter is single-tool so every experiment
is a clean repeat to average. Stable throughout on multi-world + beta 0.005.

So: a second, genuinely independent inference task trains as well as the first. The rotating-target
plumbing + multi-world batching work cleanly together.
