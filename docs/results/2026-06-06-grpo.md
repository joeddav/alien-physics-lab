# 2026-06-06 — GRPO: KL-beta tuning for stable-but-still-learning long runs

Continues the long-run stability thread from `2026-06-05-grpo.md` (the 1000-step `--vary-tools`
run learned to n≈9 / reward ~1.2 by ~step 300 then DIVERGED — entropy runaway — because
`beta=0`). The fix was a KL anchor; this doc tunes its strength.

## TL;DR
`beta` is the only real entropy/stability knob in GRPO (KL anchor to base; no entropy term in
the loss). Swept it: **`beta=0` diverges** (entropy → 8.7), **`beta=0.02` over-anchors** (stable
but plateaus ~0.63, aggregation *erodes* to ~3.5), and **`beta=0.01` is the sweet spot** — full
1000 steps, **zero divergence** (max entropy 0.77), reward **0.90**, aggregation recovered to
**n≈5–6** (still climbing at the end) with a tail to 12. So a *light* anchor stabilizes without
strangling learning. Now probing `beta=0.005` to see if even less anchor reaches the no-KL's
n≈9 while staying stable, or finally tips back into divergence (bracketing the sweet spot).

## What we learned about KL/entropy in GRPO (TRL 1.5.1)
- **Entropy is NOT in the GRPO loss.** No entropy bonus, no entropy coefficient (verified in
  `grpo_config.py`); `train/entropy` is a logged diagnostic only. The one optional hook is
  `top_entropy_quantile` (<1.0 masks the loss to high-entropy tokens; default 1.0 = off).
- The loss is `−(advantage × clipped_ratio) + β·KL(π‖π_ref)`. So **the KL term (β) is the de-facto
  entropy regulator** — indirectly, by anchoring the policy to the base (whose entropy is normal),
  preventing both collapse and runaway. With `β=0` (TRL default) nothing constrains entropy → free
  drift → the runaway we hit. β is the only direct knob for it (besides lr/lr-decay shrinking updates).
- This reframes the beta sweep: we're tuning *how hard to anchor to base*. Too hard (0.02) → drags
  aggregation back to base behavior; too soft (0) → diverges. 0.01 tests the middle.

## Note: the reward oscillation is sampling noise, not instability
The train/reward curve swings ±0.2+ step-to-step. Decomposed it (KL run, 290 steps): per-step
reward std 0.226, of which ~0.12 is within-world (only **8 rollouts of ONE seed per optimizer
step** — G8 with gen_batch=8=G → 1 world/step) and ~0.19 is between-world (each step is a
different hidden world of different difficulty). Almost all of it flows through the *physics*
(accuracy) term (std 0.20); measurement/validity are smooth. It's **stationary** (amplitude
0.25→0.20 across halves) around a *rising* smoothed trend — i.e. noise, not instability (real
instability = growing oscillation + diverging trend, like the entropy runaway). Lever to smooth it
AND lower gradient variance: **more worlds per optimizer step** (gen_batch = k×G via grad_accum),
not lr/KL. Optional, modest; deferred.

## `rl-vary-tools-g8-kl01` (beta=0.01) — the result
Full 1000 steps, exit 0. **Max entropy 0.766** over the whole run (never near the 1.5 alarm) — no
divergence. Final (last 30): reward **0.896**, physics **0.495**, entropy 0.41, tools/call **6.0**,
kl 0.35. Aggregation trajectory **n_exp 1.2 → 3.8 → 5.4** (early→mid→late), late distribution peaks
at n=4–5 (24%/24%) with a real tail (6:16%, 7:10%, 8:4%, 9:2%, 10–12:~7%), still rising at the end.
By tool subset: drop-only 3.5 / both 3.8 / pendulum-only 3.3 n_exp, accuracy ~0.37–0.41 — comparable
across regimes. So a *light* KL anchor (0.01) buys stability AND keeps ~all of the learning the
no-KL run had before it blew up. This is the working long-run recipe.
