## Validation (smoke runs)
The smoke is `--preset smoke` (Qwen3-1.7B, G=4, 8 steps). Three iterations, each fixing
a real issue — the loop ran end-to-end every time; these were tuning/correctness fixes:

| Issue | Symptom | Fix |
|---|---|---|
| Standalone vLLM on SM120 | — | cu13 `libcudart` shim + `enforce_eager`; generates correctly |
| Colocate OOM | `CUDA OOM`, 86/95 GB used | `vllm_enable_sleep_mode=True`, vLLM frac 0.45→0.30, grad-ckpt, expandable_segments |
| **Zero gradient** | `grad_norm≈1e-10`, `loss≈0`, `IS_ratio≈1e-26` | `vllm_importance_sampling_mode="token_truncate"` (was `sequence_mask`) |

After the fixes: 8/8 steps, `grad_norm≈1.9–2.95`, `reward` swinging 0.05→1.20 with
`reward_std>0` (groups have learning signal), `frac_reward_zero_std=0` on most steps.
Model thinks → runs 2–10 tool calls → `submit_answer`; `physics_reward/mean` hit 1.099
on a group where most rollouts succeeded. **Conclusion: the loop is correct and learns.**

Notes for real runs: thinking traces are long → 25–75% of completions hit the 4096-token
smoke cap (truncated → 0 reward). Real runs use a larger `max_completion_length`.
